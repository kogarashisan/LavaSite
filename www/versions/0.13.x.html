0.13.6

Feature

  • ClassManager
    • Added `Lava.ClassManager.IS_MONOMORPHIC` property (`true` by default) - in this mode generated constructors will preallocate all class properties to reduce function polymorphism. This will make construction slower, but operation will be faster. Each `_cClassData` now has `is_monomorphic` property, which indicates if it was built in monomorphic mode.
    • Added `loadClasses()` and `loadSkeletons()` convenience methods
    • Light performance improvements

Refactoring

  • ClassManager
    • RegExp objects are now moved to prototype. If you want to have unique copy of regular expression object for each class instance - then you should assign it in constructor.
    • Changed format of exported classes: now it exports both `references` and `own_references` arrays - you should delete either one of them. If you leave `references` - then you will get large, but fast and truly monomorphic class with it's own methods. In most scenarios you will want to leave `own_references` for smaller output.
    • Changed format of generated skeleton (if you did not export classes - then ignore).
    • Renamed INLINE_SIMPLE_ARRAYS -> inline_simple_arrays.
  • Fixed Lava.Cron to better work with compression
  • Fixed a bug in template rendering

Fix

  • ClassManager: fixed several serialization bugs

0.13.5

Fix

0.13.4

Feature

  • ClassManager: added possibility to store arrays in Shared

Refactoring

  • Upgraded Firestorm to v0.1.5 and used "class" property accessors from Firestorm.Element, as well as new way of referencing browser tests.
  • Removed _cClassData#level as redundant.
  • ClassManager:
    • exportClass now produces slightly less data.
    • Now it ensures that members, which are shared in parent class, are not marked as shared in inherited class.
    • Shared objects are now copied directly from class body, instead of shallow copy. May matter, if you create class instances before calling ClassManager#exportClass, otherwise ignore.

Fix

  • Serializer now correctly serializes properties with "bad" names.

Firestorm v0.1.5

  • Added class accessors like hasClass, addClass and removeClass.
  • Refactored browser compatibility checks in Environment.
