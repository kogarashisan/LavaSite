0.6.1

Feature

  • Added container.CheckboxElement and container.TextInputElement with fixes for checkbox and text inputs in IE.
  • Added public ViewManager#dispatchEvent().

Refactoring

  • Removed container.Element#setSignature().
  • Refactored ViewManager: now bubble is cancellable for all events and roles.
  • Refactored browser bugfixes in container.Element and text input widgets.

Fix

  • Fixed CheckBox widget in IE - now detects indeterminate state removal.
  • ClassManager#patch() - now will handle function removal in parent classes correctly.
  • lava-framework.com: fixed CheckTree example.

Build contains other minor refactorings.

0.6.0

Feature:

  • Foreach view now has "depends" hash option for Lava.scope.Foreach - to allow sorting and filtering inside the scope.
  • Added ClassManager#patch() - allows replacing class methods in any point of hierarchy (to apply browser fixes).
  • Added _cClassData#hierarchy_names - short names of classes.

Refactoring

  • Minor refactoring of Lava.init and Lava.bootstrap: PopoverManager and FocusManager will be created only if there is class in schema. Now they both have enable(), disable() and isEnabled().
  • Renamed Accordion option: keep_expanded_on_add -> keep_new_panels_expanded.
  • Accordion sugar now does not require the <panels> tag inside it.
  • Removed "cancel_refresh_on_input" option from text inputs.
  • Changed template of TextInput widget.
  • Added PasswordInput widget (no differences from TextInput, but applies fixes for IE8-9).
  • ClassManager now does not allow duplicate short class names in inheritance chain.
  • Renamed Input/Abstract.class.js -> Input/InputAbstract.class.js

Fix

  • ExpressionParser now will ignore switch-like sequences ("{test:}", "{:test}").
  • Fixed crash when parsing "storage_object" sugar with directives.
  • Fixed toQueryString in Checkbox and Radio widgets.
  • Fixed text input widgets in IE9 - now they will refresh their value, when "backspace" or "del" keys are pressed (IE-related bugs).

Build also contains minor improvements and refactorings
