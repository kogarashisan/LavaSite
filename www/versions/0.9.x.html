0.9.0

Reworked the Scope and View layers. Now Lava is even faster then before.

Certain limitations on refresh cycles were also removed: now you can manipulate data while views are refreshing.

Feature

  • Added _cRefresher#insert_callback, remove_callback, and removal_strategy.
  • Improved a couple of parser error messages.
  • Added Firestorm.String#escapeStringForRegExp().
  • Added MetaStorage#createMetaRecord().
  • Tree widget now can insert and remove nodes without animation interruptions and redrawing of entire content.
  • Added Lava.view.container.Element#getStartElement(), getEndElement(), which return it's DOM element - for compatibility with Morph container.

Refactoring

  • Removed "waits_refresh" and "refreshed" events from Scope layer, now they are aligned by Refreshable#level property.
  • Removed broadcastSleep() and broadcastWakeup() functionality from views, as well as isSleeping().
  • MetaStorage#get does not create records automatically anymore.
  • MetaRecord now inherits from Lava.data.Record and has `_original_record` property.
  • Removed Lava.data.RecordAbstract, merged into Lava.data.Record.
  • Lava.widget.Standard#set() now has equality check. Custom setter is called only if new value differs. Logic in widget property setters was simplified.
  • Renamed ScopeManager.refreshScopes() -> ScopeManager.refresh().
  • Tree widget was completely rewritten (including template). Options format has changed: now instead of old "use_meta_storage" option you must specify {meta_storage_columns: ["is_expanded"]}.
  • Refactored internal structure and API of refreshers.

Fix

  • Fixed a series of bugs in view and scope layers, that could crash the application.
  • Now roles can be dispatched inside events and vice versa.

Site

  • Added new examples
