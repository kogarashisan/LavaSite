0.4.0

Due to nature of "alpha" version, build contains lots of undocumented bug fixes, refactorings, changes and improvements. But keep in mind: since "beta" version - every change will be described in the change log. Beta is coming soon.

Feature

  • Added "dynamic scopes".
  • Declarative template editing (via edit_template tag in widget definition).
  • Added insertHTMLBefore() to all containers.
  • Emulated container: rewritten the insertion algorithm. New config options available, old gone.
  • Added Animator/Color.class.js
  • Animation now uses requestAnimationFrame()

Refactoring

  • Refactored the template of the Tree class, mainly for use of dynamic scopes.
  • Most classes, that implemented mixins - now extend them.
  • (container.Element) wrap() was split into 2 methods: renderVoid() for void containers, and wrap(html) for non-void.
  • Renamed in Enumerable:
    • remove -> removeValue
    • include -> includeValue
    • contains -> containsValue
  • Tooltip widget: now does not escape html. PopoverManager: replaces newlines with <br>
  • Widget definitions: <view role="..."> and <template role="..."> are split into 3 new tags: <include>, <main_view> and <main_template>.
  • Completely rewritten Sugar class, and all _cWidget.storage functionality. Added separate schema for storage.
  • Tabs widget: added 'is_active' property to each tab, refactored.
  • Animation: renamed config options. 'transition' -> 'transition_name', and 'transition' is now a function.
  • All widget setters now accept arguments in different order: _setSomething(value, name)
  • Standard widget now has a name: 'widget'
  • Enumerable.each(): changed the callback parameters order. New order: value, name, uid, index.

Fixes

  • Fixed a serialization issue in Lava.Serializer.
  • Fixed a bug in view refresh system.
  • Fixed a bug in template compiling algorithm.
  • Fixed a bug in resources merging, which created duplicate classes.
  • Tooltip now hides empty tooltips by default.
  • Enumerable: fixed refresh from array.

0.3.0

Feature:

  • Foreach view now supports sorting and filtering of it's values, via external callback
  • Calendar widget now prevents text selection when clicking on a day
  • ExpressionParser now supports dependencies for expressions
  • ExpressionParser: generated argument configs are more compact
  • Parent includes can now be included in inherited widgets

Refactoring:

  • ObjectParser: allowed external paths now must start with 'Lava'
  • Renamed Lava.data.Enumerable -> Lava.system.Enumerable
  • Changed signature of Enumerable filters
  • Enumerable now does not support .get(index), as well as "property changed" and index_changed events for indices. The only available property in Enumerable now is 'length'.
  • Lots of refactorings in Data layer, added Guid field
  • Changed format for Argument config and many other minor changes

Fixes:

  • Animation class now properly generates it's GUIDs
  • TemplateParser now correctly parses the <script> tags
  • TemplateParser: now concatenates adjacent strings in many cases
  • Fixed some bugs in Collection and Record fields
  • Fixed "Debug assertion failed" in "If" view
