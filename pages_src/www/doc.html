Introducing LiquidLava

Lava is an MVC framework with unique architecture: everything here is a class, or at least a JavaScript object. Lava has excellent sources, so if something is unclear to you - then sources is the first place, where you should search for answers.

So, what is LiquidLava?

You may think of it as better alternative to modern frameworks with data-bound templates, like Angular and Ember. Everything you can do with these frameworks - you can do with Lava, except that Lava does it better. So, if you are tired of endless problems and bugs with other frameworks - consider trying Lava instead, you will feel a relief and boost to your productiveness.

Project status?

Lava is already mature enough to be comfortably used in small projects. Lack of unit tests is compensated by numerous debug checks, API is mostly stable.

Currently it lacks some high-level features, like server-side communication or routing, and that's the reason, why it's still considered alpha. Some of missing features can be replaced by third-party scripts, but even now it's powerful enough to impress you.

Introducing Firestorm

Firestorm is a low-level DOM access library, currently built on top of MooTools. It does not extend prototypes of native JavaScript objects - this makes it extremely simple and reusable. In fact, it's a collection of JavaScript objects with public API.

Any access to DOM tree must be done using Firestorm API (although, there are exceptions).
